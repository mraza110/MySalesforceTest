CREATE A BRANCH:

  • Now a good prac is to always create a branch.
  • To do that use the command git branch branchname so git branch dev
  • Created a branch in working environment
  • Than do git checkout dev -m”This is a dev branch”
  • Commit the branch to staging
  • Than do git push --set-upstream origin dev
  • Push the branch to remote Repository
  • Go to github and check .. under master you should see ur branch.
  • A short form to this process is using the command git checkout -b my_branch
  • Than go git push --set-upstream origin my_branch

This is my little paragraph about the branching strategy. These are the salient principles for us to follow:

  • master should always store what exactly we have in production.
  • dev should be the in-development codebase the dev team uses.
  • feature branches should be created off of dev for any feature you work on.
  • release branches should be created off of dev for any releases.
